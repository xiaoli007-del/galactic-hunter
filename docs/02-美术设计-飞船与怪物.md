# 银河猎手 — 美术系统设计(飞船与怪物)

| 项 | 内容 |
|---|---|
| 文档版本 | v0.11(2026-06-30) |
| 定位 | 2D / 伪 3D 程序化 Canvas 美术系统,支持「升级外观变化 + 多敌人类型」 |
| 实现 | `src/js/render.js`(运行时按等级/类型组装模块,非静态贴图) |

> **为什么是程序化 Canvas 而非 PNG 贴图**:升级外观变化要求飞船随等级动态变形(机翼展开、推进器增加、能量核心增强),静态 PNG 无法表达。Canvas2D 的渐变 / 合成 / 路径远强于扫描线光栅化,且不依赖任何二进制图片资源 —— 沿用项目「避开图像生成/识别报错」的哲学。`assets.js` / `gen-sprites.js` 保留不破坏既有工作流,但 render 层已改用更强的程序化绘制为主路径。

## 一、玩家飞船(模块化 + 阶段进化)

飞船由 5 个模块组成,按 `ship.level`(1–5)逐级组装。所有模块在 `render.js` 的 `ship()` 中以本地坐标(原点=飞船中心,旋转 `aimAngle`)绘制,统一乘 `visScale`(体型随等级放大,碰撞半径不变)。

### 1.1 模块拆分(结构建议)

| 模块 | 方法 | 作用 | 进化规则 |
|---|---|---|---|
| 推进器 Engine | `_shipEngine` | 喷口 + 尾焰 | 喷口数 1→2→3→4→5,尾焰随等级变长变亮 |
| 机翼 Wings | `_shipWings` | 左右机翼 | 窄→宽→分叉(Lv3+)→能量翼缘(Lv4+) |
| 船体 Hull | `_shipHull` | 中心船体骨架 | 梭形→装甲条纹(Lv2+)→分体结构(Lv4+) |
| 武器挂载 Weapons | `_shipWeapons` | 炮管 | 1→2→3→4 管,高级多联装(Lv3+) |
| 副炮座 Turrets | `_shipTurrets` | 舰载自动副炮(v0.10) | Lv2-3=1门、Lv4-5=2门;Lv2 固定朝上、Lv3+ 自动锁敌 |
| 能量核心 Core | `_shipCore` | 中心发光体(**最关键**) | 尺寸/亮度随等级增强,Lv5 悬浮 + 能量环 |

绘制顺序(后到前):引擎 → 机翼 → 船体 → 武器 → 副炮座 → 能量核心。

### 1.2 各等级外观描述(可用于建模 / AI 绘图参考)

> 配色:船体金属 `#2a3340`→`#aebccf` 渐变;能量核心与光晕用 `Config.SHIPS[].glow`(Lv1 青→Lv5 品红)。

**Lv1 侦察艇 Scout**
- 小型梭形船体,单推进器,弱发光核心(青色,小而暗)
- 窄短机翼,1 根炮管
- 轻量金属质感,结构单一
- 视觉关键词:轻、快、入门

**Lv2 驱逐舰 Frigate**
- 机翼展开(更宽),增加副推进器(双喷口)
- 船体增加护甲板(横向条纹)
- 武器 1→2,核心变亮
- 视觉关键词:改装、战斗艇

**Lv3 巡洋舰 Cruiser**
- 明显「装甲块拼接结构」(多条分割线)
- 机翼分叉(双层/分裂),3 推进器
- 能量核心外露、发光增强,3 炮管多联装
- 视觉关键词:重型、突击

**Lv4 战列舰 Battleship**
- 船体变大 + 分体结构(中线缝隙)
- 4 推进器,机翼带能量翼缘(发光边)
- 核心强发光,4 炮管
- 视觉关键词:旗舰级、危险

**Lv5 旗舰 Dreadnought**(终极形态)
- 船体最大 + 分体结构,5 推进器环形布局
- 能量核心**悬浮在中心**(脉动浮动)+ 能量环
- 能量翼缘光效最强,4 炮管多联装
- 视觉关键词:母舰、终极、科技感拉满

### 1.3 升级表现规则(强制)

升级必须「看起来变强」,不能只换皮:
1. **体积逐级增加**(`visScale = 1 + (lvl-1)×0.06`)
2. **武器数量增加**(1→2→3→4)
3. **发光强度增强** —— 核心最关键(尺寸 + alpha 随等级增长)
4. **结构 单体 → 拼装 → 机械复杂化**(条纹→分体→悬浮)
5. **推进器尾焰逐级变长**(喷口数 + 焰长)
6. 整体越来越「危险 / 科技感」

## 二、怪物设计系统(多类型生态)

6 种 tier,每种独立轮廓 + 发光弱点核心,一眼可辨类型;复杂度 / 光效随 tier 递增;统一「太空机械 / 生物机械混合」风格。配色见 `Config.ALIENS[].color`。

### 2.1 统一规则
- 都有**发光弱点核心**(中心径向亮光 + `drawGlow` 光晕;Boss 为多核心节点)
- 形状**一眼区分**类型(水滴 / 三角翼 / 六边形 / 波浪体 / 八边形 / 巨构团块)
- 越强 → 结构越复杂 + 光效越强
- 太空机械 / 生物机械混合风格

### 2.2 各类型设计说明

| Tier | 类型 | 轮廓 | 特征 | 玩法定位 |
|---|---|---|---|---|
| t1 | 追踪虫 Drone Bug | 装甲水滴体 | 三尖刺腿 + 上颚 + 小亮核 | 快速 / 自爆感,大量刷新 |
| t2 | 拦截机 Interceptor | 后掠三角翼 | 双引擎光 + 中心核 | 中距射击型,常见 |
| t3 | 装甲巨兽 Armored Brute | 六边形装甲 | 双钳臂 + 凹陷核心 | 高防肉盾,慢 |
| t4 | 相位幽影 Phase Wraith | 半透明波浪体 | 触须 + 闪现回避 | 闪避型(受击概率闪现) |
| t5 | 精英装甲 Elite Armor | 不规则八边形机械体 | 肩炮 + 装甲分割 + 暴露弱点核 | 高分目标,大 |
| t6 | 巨构 Boss Colossus | 分段团块(占屏 1/3) | 多核心节点 + 顶刺 + 装甲段 | 稀有,击杀成就感 |

### 2.3 实现

- `alien()`:外光晕 → 径向装甲渐变本体 → `switch(tier)` 调对应轮廓方法 → 发光弱点核心(非 Boss)→ 状态叠加(冰冻/灼烧)→ 血条
- 各 tier 轮廓方法:`_alienCrawler` / `_alienDrone` / `_alienBrute` / `_alienWraith` / `_alienElite` / `_alienBoss`

## 三、整体美术风格

- **科幻 + 发光能量风**:深空背景 + 高对比霓虹光(青/品红/金/橙)
- **轮廓清晰**:适合手游小屏辨识
- **Arcade 射击风格**(雷电 + 现代科幻)
- **性能**:发光用预渲染精灵(`drawGlow`,按色缓存复用),避免 `shadowBlur`(Canvas2D 头号性能杀手)
- **受击反馈**:本体闪白(`hitFlash`)+ 冰冻冷光圈 / 灼烧橙光脉动(`_alienStatus`)

## 四、扩展建议

## 六、v0.10 船舰副炮养成实装(2026-06-29)

补做 GDD §4.3 写明但此前未实装的「炮位」—— 船舰此前只有 `fireMul` 乘区,缺实质玩法价值。副炮是**船舰自带的自动火力点**,随舰级解锁门数 / 模式,提供持续辅助 DPS,让船舰养成有体感反馈。

### 6.1 炮位阶梯(贴合 GDD §4.3)

| 舰级 | 副炮门数 | 模式 | GDD 对应 | 数值(turretDmg / turretRate) |
|---|---|---|---|---|
| Lv1 侦察艇 | 0 | — | — | — |
| Lv2 驱逐舰 | 1 | 固定朝上 | 双炮位 | 1 / 2.0 |
| Lv3 巡洋舰 | 1 | 自动锁敌 | +自动副炮 | 2 / 2.5 |
| Lv4 战列舰 | 2 | 自动锁敌 | 三炮位 | 2 / 3.0 |
| Lv5 旗舰 | 2 | 自动锁敌+强化 | 全向射击 | 3 / 3.6 |

### 6.2 设计要点

- **独立伤害、不乘 fireMul**:副炮是舰载固有火力,乘区已在主武体现(`fire(w)` 里 `w.damage * fireMul`);副炮 `turretDmg` 独立,避免双计。
- **纯物理、不吃技能**:`skill=null`,不受 `activeSkill`(冰/火/电/激光/散射)影响 —— 副炮是稳定的兜底输出,主武承担技能变化。
- **复用 Bullet 实体**:副炮弹构造为普通子弹后覆盖 `color`(亮绿 `#9bff7a`)/ `radius`(3.6,细于主武 5)/ `pierce`(0,不贯穿)/ `turret=true`;`render.bullet` / 碰撞链路零特判,自动走通。
- **自动锁敌用 `Ship._findTarget`**:v0.7 移除主武自动锁敌时保留未用的方法,现由副炮启用(GDD §4.3「全向射击」);无目标退回朝上。

### 6.3 美术(_shipTurrets)

- 副炮座画在机翼前缘(`y=0.12r`,避开顶部主炮管 `-1.3r` 与驾驶舱 `-0.55r`)
- 结构:金属梯形底座 + 短炮管(朝上)+ 炮口环(深)+ 炮口蓄能芯(发光,副炮弹色;Lv3+ 自锁略亮)
- 1门居中 / 2门机翼左右对称(`±0.5r`)
- 炮塔座**固定朝上画**,自动锁敌由子弹方向体现(不旋转模型,简化实现)

### 6.4 验证

- `node tools/smoke.js` → 186/0(+20:配置阶梯 / 自动开火 / 弹道属性 / 自锁方向 / 击杀链路 / 渲染路径)
- HUD:底栏船舰卡新增「副炮×N [自锁]」状态行

## 七、v0.11 5 新 Boss + 机制系统(2026-06-30)

在 v0.10 系列(AI 生图贴图接入:飞船/子弹/敌弹/爆炸/胶囊/旧 3 Boss 按 type 名自动接入)基础上,新增 **5 个高阶 Boss**(泰坦/九头蛇/晶体/巨口/霸主),每个配一个鲜明**机制**(shield/summon/absorb/wave),`bossRotation` 扩到 8 Boss 轮换;并加副炮小弹贴图 + 流动背景视差,强化后期压迫感与飞行体感。

### 7.1 5 新 Boss 清单

| Boss | type | tier | hp | 机制 mechanism | bossVisScale | 阶段弹幕(1→2→3) |
|---|---|---|---|---|---|---|
| 泰坦战舰 | boss-titan | 8 | 1500 | shield(护盾吸收) | 1.88 | aimed→aimed→ring |
| 九头蛇母舰 | boss-hydra | 9 | 1800 | summon(召唤 t7 仆从) | 1.60 | spiral→spiral→ring |
| 晶体巨核 | boss-crystall | 9 | 2200 | absorb(吸收伤害回血) | 1.39 | ring→spiral→ring |
| 虚空巨口 | boss-maw | 10 | 2800 | wave(位移冲击波) | 1.55 | spiral→wave→ring |
| 霸主中枢 | boss-overlord | 10 | 3500 | summon+shield | 1.45 | aimed→spiral→ring |

- `spawnWeight: 0` —— 不进普通刷新池,仅由 `WAVE.bossRotation` 8 序列循环触发(t6/t9/t10 → 5 新 Boss → 回 t6)
- 每个 Boss 贴图按 type 名自动接入(`getAlienTex(a.type)`,type 名即贴图名),缺失退回程序化矢量

### 7.2 四种机制

- **shield**(titan/overlord):Boss `shield>0` 时 `takeDamage` 扣盾不扣血、返回 `'shield'`;盾打穿前 hp 不降。`_onBossStage` 阶段 2/3 补满护盾(需先破盾才能扣血)。渲染:护盾能量环 + 盾条(血条上方)
- **summon**(hydra/overlord):阶段 2/3 `_bossSummon` 用 per-Boss `summonType`/`summonCount`(hydra→t7×3 / overlord→t8×2),复用既有召唤链路
- **absorb**(crystall):`takeDamage` 在 hp<maxHp 时 30% 概率把伤害转为回血(dmg×0.5)、返回 `'absorb'`;飘字「+N 吸收」
- **wave**(maw):阶段≥2 锚点 X 正弦横移压迫(阶段3 振幅更大);阶段 2 发 `wave` 冲击波弹 —— 全圆均布、`expand=true` 半径膨胀限 40、速度衰减模拟扩散
- 飘字按命中类型区分:护盾「护盾 N」/ 吸收「+N 吸收」/ 普通「-N」

### 7.3 其他改动

- **副炮子弹贴图** `turret-bullet`:`render.bullet()` 按 `b.turret` 选图(此前副炮弹无贴图走矢量退回)
- **流动背景视差**:`background(ctx,t,dt)` bg.png 双图上下平移无缝循环(40px/s)+ 星点按 layer 下沉回卷,营造飞船持续向上飞行感
- 调试键 1-5 切飞船等级 + 金币拉满仍可用
